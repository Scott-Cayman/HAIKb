# 企业知识云盘系统 - 功能模块说明

## 📋 项目概述

**企业知识云盘系统 (Enterprise Knowledge Drive)** 是一个基于 FastAPI + React 的企业级知识库管理系统。该系统不仅提供传统的云盘存储功能，还集成了基于 AI 的 RAG (Retrieval-Augmented Generation) 检索增强能力，能够自动生成文档摘要并支持智能问答。

---

## 🎯 核心设计理念

1. **分层存储**: 原文件负责资产沉淀，AI 摘要负责索引和检索
2. **限制范围**: 只解析文档前 10 页生成摘要，防止全文索引导致的 token 消耗
3. **参考 kotaemon**: 借鉴了 kotaemon 项目的 RAG 索引管理设计思路
4. **双模式**: 同时支持传统文件管理和 AI 智能检索两种使用方式

## 📦 功能模块详解

### 1. 用户认证与权限模块 (Authentication & Permissions)

#### 1.1 核心功能
- **模拟登录**: 支持管理员和普通用户两种模拟账号快速登录
- **JWT Token 认证**: 使用 JWT 进行会话管理和身份验证
- **钉钉登录预留**: 预留了钉钉 OAuth2 登录接口，可后续接入
- **权限路由保护**: 前台和后台使用不同的路由和权限校验

#### 1.2 API 接口
- `POST /api/auth/login-mock-admin` - 模拟管理员登录
- `POST /api/auth/login-mock-user` - 模拟普通用户登录
- `GET /api/auth/me` - 获取当前用户信息

#### 1.3 数据模型
- `User` - 用户表，包含基本信息和管理员标识

---

### 2. 文件夹管理模块 (Folder Management)

#### 2.1 核心功能
- **层级文件夹**: 支持多级嵌套的文件夹结构
- **文件夹创建**: 支持新建文件夹和相对路径自动创建
- **文件夹查看**: 展示文件夹树和文件夹详情
- **软删除**: 删除操作使用软删除标记，数据可恢复

#### 2.2 API 接口
- `GET /api/folders/` - 获取根级文件夹列表
- `GET /api/folders/tree` - 获取完整文件夹树结构
- `GET /api/folders/{folder_id}` - 获取文件夹详情
- `POST /api/folders/` - 创建新文件夹
- `PATCH /api/folders/{folder_id}` - 更新文件夹信息
- `DELETE /api/folders/{folder_id}` - 删除文件夹

#### 2.3 数据模型
- `Folder` - 文件夹表，包含父子关系、创建者等
- `FolderPermission` - 文件夹权限表（预留扩展）

---

### 3. 文件管理模块 (File Management)

#### 3.1 核心功能
- **文件上传**: 支持单文件上传，支持拖拽上传
- **相对路径支持**: 上传时可指定相对路径，自动创建层级文件夹
- **文件下载**: 原文件下载，支持下载计数统计
- **文件预览**: 
  - PDF、图片直接预览
  - Word/PPT 自动转换为 PDF 预览（依赖 LibreOffice）
- **文件更新**: 支持重命名等基本操作
- **文件删除**: 软删除，支持查看计数和下载计数
- **最近文件**: 展示最近上传的文件列表
- **标题搜索**: 基于文件名的快速匹配搜索

#### 3.2 API 接口
- `POST /api/files/upload` - 上传文件
- `GET /api/files/recent` - 获取最近文件
- `GET /api/files/folder/{folder_id}` - 获取文件夹内文件
- `GET /api/files/title-search` - 标题搜索
- `GET /api/files/{file_id}` - 获取文件详情
- `GET /api/files/{file_id}/download` - 下载文件
- `GET /api/files/{file_id}/preview` - 预览文件
- `PATCH /api/files/{file_id}` - 更新文件信息
- `DELETE /api/files/{file_id}` - 删除文件

#### 3.3 数据模型
- `File` - 文件表，包含存储路径、预览路径、统计计数等

#### 3.4 文件转换流程
```
上传 Word/PPT
    ↓
后台任务启动
    ↓
调用 LibreOffice 转换为 PDF
    ↓
更新 preview_path 和 preview_status
    ↓
预览可用
```

### 4. RAG 摘要与索引模块 (RAG Summary & Indexing)

#### 4.1 核心功能
- **文档解析**: 提取文件前 10 页的文本内容
- **AI 摘要生成**: 调用 LLM 生成结构化的 AI 摘要
- **摘要格式**: 
  - 一句话判断
  - 两句话简介
  - 结构化标签（客户类型、项目类型等）
  - 重要信息摘要
  - 可复用价值
  - 推荐检索问题
  - 检索关键词扩展
- **摘要切片**: 将生成的 Markdown 摘要切分为适合向量检索的 Chunk
- **向量存储**: 支持向量索引存储（预留多种适配器）
- **关键词索引**: 支持关键词检索，与向量检索混合使用
- **索引重建**: 支持批量重建索引

#### 4.2 API 接口
- `GET /api/rag/indices` - 获取 RAG 索引列表
- `POST /api/rag/indices/default/rebuild` - 重建默认索引
- `POST /api/rag/files/{file_id}/summarize` - 手动触发摘要生成
- `GET /api/rag/files/{file_id}/summary` - 获取文件摘要
- `PUT /api/rag/files/{file_id}/tags` - 更新摘要标签（管理员）
- `POST /api/rag/files/{file_id}/reindex-summary` - 重新索引摘要
- `GET /api/rag/status` - 获取 RAG 系统状态

#### 4.3 数据模型
- `DocumentSummary` - 文档摘要表，存储 AI 生成的摘要和标签
- `RagIndex` - RAG 索引配置表
- `RagSource` - RAG 源记录表，关联摘要和索引
- `SummaryChunk` - 摘要切片表，存储切分后的文本块
- `RagIndexRelation` - 索引关系表，维护源与切片/向量的关系

#### 4.4 摘要生成流程
```
文件上传完成
    ↓
后台任务触发
    ↓
提取前 10 页文本
    ↓
调用 LLM 生成 Markdown 摘要
    ↓
保存到 DocumentSummary
    ↓
Markdown 文件保存到 storage/summaries/
    ↓
进入 RAG 索引流程
    ↓
摘要切片
    ↓
向量化
    ↓
存储到向量库和 SummaryChunk
    ↓
更新索引状态
```

### 5. Agent 智能问答模块 (Agent Q&A)

#### 5.1 核心功能
- **自然语言提问**: 用户用自然语言描述需求
- **混合检索**: 向量检索 + 关键词检索结合
- **检索增强生成**: 基于检索到的摘要片段生成回答
- **相关文件推荐**: 自动推荐匹配的原文件
- **推荐信息**: 每个推荐文件包含两句话简介和推荐理由
- **对话历史**: 支持 conversation_id 跟踪对话上下文

#### 5.2 API 接口
- `POST /api/agent/chat` - Agent 对话接口
  - 参数: query, conversation_id, top_k, retrieval_mode
  - 返回: answer, evidence, related_files

#### 5.3 数据模型
- `AgentMessage` - Agent 消息表，记录对话历史

#### 5.4 Agent 问答流程
```
用户输入问题
    ↓
问题向量化
    ↓
向量检索 (Vector) + 关键词检索 (Keyword)
    ↓
合并检索结果，去重重排
    ↓
提取相关 Chunk 作为 Evidence
    ↓
构建 Prompt，包含 Evidence
    ↓
调用 LLM 生成回答
    ↓
关联原文件，生成推荐列表
    ↓
返回给前端展示
```

## 📈 关键业务流程

### 流程 1: 文件上传 → 自动摘要 → 索引
```
1. 用户上传文件
   ↓
2. 保存原文件到 storage/originals/
   ↓
3. 如果是 Office 文档，启动 PDF 转换任务
   ↓
4. 启动摘要生成任务 (generate_summary_and_index_task)
   ↓
5. 解析前 10 页文本
   ↓
6. 调用 LLM 生成 Markdown 摘要
   ↓
7. 保存到 document_summaries 表
   ↓
8. 保存 Markdown 文件到 storage/summaries/
   ↓
9. 摘要切片 + 向量化
   ↓
10. 存储到 summary_chunks 和向量库
    ↓
11. 更新 file.summary_status = "success"
```

### 流程 2: Agent 问答
```
1. 用户在搜索页输入问题
   ↓
2. 前端调用 POST /api/agent/chat
   ↓
3. 问题预处理 + 向量化
   ↓
4. 混合检索 (Vector + Keyword)
   ↓
5. 合并结果，重排 Top-K
   ↓
6. 构建 Evidence Context
   ↓
7. 调用 LLM 生成回答
   ↓
8. 从检索结果关联原文件
   ↓
9. 组装 related_files（包含两句话简介）
   ↓
10. 返回给前端
    ↓
11. 前端展示回答 + 推荐文件卡片
    ↓
12. 用户可点击卡片预览原文件
```

## 🚀 启动方式

### 后端启动
```bash
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --reload
```
- 服务地址: `http://localhost:8000`
- API 文档: `http://localhost:8000/docs`

### 前端启动
```bash
cd frontend
npm install
npm run dev
```
- 服务地址: `http://localhost:5173`
